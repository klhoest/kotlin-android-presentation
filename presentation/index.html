Introduction

Kotlin est

  • Orienté objet et fonctionnel
  • Typage statique
  • Conçu pour s'executer sur la JVM (Android)..
  • ..mais aussi pour être compilé sur Linux et convertit en Javascript
  • mélangé avec du code Java et les bibliothèques Java
  • Peut fonctionner avec Java 6+
  • Créé par JetBrains

Les apports du Kotlin par rapport au Java

  • Lambda expressions + Inline functions
  • Extension functions
  • Null-safety
  • Smart casts
  • String templates
  • Properties
  • Primary constructors
  • First-class delegation
  • Data classes
  • Type inference for variable and property types
  • Declaration-site variance & Type projections
  • Range expressions
  • Operator overloading
  • Companion objects
  • Separate interfaces for read-only and mutable collections
  • Coroutines

// c# 8
interface Field { 
	fun validate(): String 
}
class EmailField: Field{ 
	override fun validate(): String = "" 
}

abstract class Form {
	protected val properties: MutableMap = mutableMapOf()

	fun validate(): Collection {
		val errors = arrayListOf()
		for (field in properties) {
			errors.add(field.value.validate())
		}
		return errors
	}
}

class OrderForm: Form() {
	var email: EmailField by properties
	init {
		email = EmailField()
	}
}

Higher-Order Functions

A higher-order function is a function that takes functions as parameters, or returns a function 
fun lock(lock: Lock, body: () -> Unit): Unit {
	lock.lock()
	try {
		return body()
	}
	finally {
		lock.unlock()
	}
}
fun toBeSynchronized() = sharedResource.operation()
val result = lock(lock, { sharedResource.operation() })
